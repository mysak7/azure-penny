azure-penny

A serverless Azure cost-visibility dashboard.
Reads Cost Management exports from Blob Storage
and serves interactive dashboards via FastAPI on Container Apps.

FastAPI Azure Container Apps Blob Storage Managed Identity Terraform Scale to Zero
The Problem It Solves
Azure billing is powerful but opaque — azure-penny makes it instant and actionable
🔍
Cost Visibility
Azure Cost Management exports daily CSV files to Blob Storage. azure-penny aggregates them into a live, filterable dashboard — no portal login needed.
🏗️
Resource Group Drill-down
Filter costs by any resource group, time period (day / week / month), and service category. Spot which team or project is driving the bill.
⚡
Live Resource Inventory
A second tab lists every live Azure resource, its monthly cost, and potential spot-price savings — cross-referenced with the export data.
🗑️
One-click Cleanup
Delete individual resources or entire resource groups directly from the UI — streaming real-time progress back over HTTP.
💰
Zero Idle Cost
Container App scales to zero when nobody is looking. A cost dashboard that costs nothing when you're not using it.
🔒
No Secrets, Ever
Managed Identity + DefaultAzureCredential — no connection strings, SAS tokens, or service principal secrets in the codebase or CI.
High-Level Architecture
Everything inside one Azure resource group, managed by Terraform
Infrastructure — Terraform
All resources declared in terraform/ · State stored remotely in Azure Blob

Key Design Decisions
  • 1Scale to zero — min_replicas=0, max_replicas=1. Cold-start is acceptable for a dashboard checked once or twice a day.
  • 2Single revision mode — one active revision at a time. Blue/green is unnecessary for this workload.
  • 3LRS replication — exports are re-created daily; geo-redundancy would be wasted spend.
  • 4Blob versioning + soft delete (7d) — guards against accidental deletion at negligible cost.
  • 5CSV export format — the azurerm provider doesn't support Parquet at subscription scope; app handles both for forward compatibility.

Authentication — Zero Secrets
User-Assigned Managed Identity + DefaultAzureCredential everywhere · No passwords, ever
Why Not a Service Principal Secret?
  • ✗Secrets rotate, expire, and leak — introducing a full secret-management lifecycle.
  • ✗SAS tokens expire and hard-coded tokens are a security risk.
  • ✗Connection strings give full storage access — over-privileged by default.
  • ✓Managed Identity removes the secret-management lifecycle at zero extra cost. Works identically in ACA, local dev, and CI/CD.

Export Path Convention
{export-name}/ {YYYYMMDD-YYYYMMDD}/ ← date range folder {guid}/ {filename}.csv ← cumulative file
Exports are cumulative within a billing period — each run overwrites or adds a new file in the same date-range folder. Reading multiple files from the same folder would multiply costs, so only the file with the latest last_modified timestamp is loaded.

CI/CD — Three GitHub Actions Workflows
OIDC authentication · No stored secrets in the repository
No Stored Secrets
GitHub variables hold only non-secret IDs (client_id, tenant_id, subscription_id). OIDC federated credentials eliminate password rotation.
Production Gate
CD and Terraform apply require the production GitHub environment — enabling manual approval if configured.
Image Tagging
Every push creates a new image tagged with both github.sha and latest — enabling rollback to any commit.

Deploy & Start Saving
From zero to live cost dashboard in three commands
1. Bootstrap Terraform state backend
bash scripts/bootstrap-tfstate.sh
2. Provision infrastructure
cd terraform terraform init terraform apply \ -var="container_image=placeholder"
3. Push code → CI/CD does the rest
git push origin main # → CI validates # → CD builds + pushes image to ACR # → deploys to Container App
Cost Management will start exporting on its daily schedule. Force an early check with POST /api/reload.
