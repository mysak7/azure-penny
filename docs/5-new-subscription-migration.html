Cost Export — Three-Act Debugging
From "not supported" to a working daily export in billing profile scope
🧪 Act 1 — Free Trial
Subscription quota was FreeTrial_2014-09-01. Azure Cost Management exports are completely unsupported on free trial — every API version returns 404.
Added enable_cost_export = false variable
⚙️ Act 2 — Upgraded to PAYG
Quota changed to PayAsYouGo_2014-09-01. azurerm_subscription_cost_management_export still fails — the provider uses newer API versions that the MCA billing backend rejects at subscription scope.
Provider API version mismatch
✅ Act 3 — Billing Profile Scope
For MCA accounts, exports must target the billing profile scope, not the subscription. Replaced azurerm_subscription_cost_management_export with azapi_resource at the billing profile scope.
✓ Export Active · Daily CSV
Cost Export Fix — azapi to the Rescue
azapi_resource lets us control the exact scope and API version
Before — azurerm (broken)
resource "azurerm_subscription_cost_management_export" "daily" { # Uses newer API rejected by MCA billing # scope = subscription (wrong for MCA) name = "penny-prd-daily-export" subscription_id = data.azurerm_subscription.current.id ... }
After — azapi (working)
resource "azapi_resource" "cost_export" { type = "Microsoft.CostManagement/exports@2023-11-01" name = "penny-prd-daily-export" # billing profile scope — correct for MCA parent_id = var.billing_profile_id body = { properties = { schedule = { recurrence = "Daily", ... } definition = { type = "ActualCost", ... } deliveryInfo = { destination = { resourceId = azurerm_storage_account.this.id container = "cost-exports" } } } } }
